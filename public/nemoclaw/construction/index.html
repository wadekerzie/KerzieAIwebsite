01

NemoClaw Platform Installation

Deploy NemoClaw into your chosen environment — cloud, on-prem VM, or Kubernetes. Configure storage, logging, and model backends so agents run reliably. Basic security hardening included.

  • Cloud, on-prem, or Kubernetes deployment
  • Storage, logging, model backend config
  • TLS, secrets management, access controls
  • Network exposure hardening
02

Identity & Access Integration (SSO)

Connect NemoClaw to your identity provider so agents are treated like users — assigned to groups/roles, with permissions following those roles. IT can onboard/offboard agents the same way as employees.

  • Azure AD, Okta, Google Workspace, or internal SSO
  • Agents assigned to groups and roles
  • Permissions follow role assignments
  • IT onboards/offboards agents like employees
03

Messaging Integration

Integrate NemoClaw with your primary collaboration tools so staff can DM agents directly, mention them in channels, and treat them as coworkers in existing communication flow.

  • Slack, Microsoft Teams, or Google Chat
  • Agent endpoints for direct messaging
  • Channel mentions and conversations
  • Agents as coworkers in your existing flow
04

API & Tool Wiring

Connect NemoClaw to the key systems your agents will touch — dispatch, job management, ERP, CRM, file storage. Define tool adapters so agents can read data and draft actions.

  • Dispatch / field service platform
  • Job management / ERP / accounting
  • CRM, email, file storage
  • Tool adapters for read + controlled write
05

Sandbox & Guardrails

Define the safe field your agents can play on — which systems are read-only vs write-capable, which actions require human approval, what data agents can access. Full logging and audit.

  • Read-only vs write-capable system boundaries
  • Human-approval gates for sensitive actions
  • Data access categories defined
  • Full activity logging and audit trail
